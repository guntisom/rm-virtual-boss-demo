Key Risks & Mitigations
CRM Data Quality
High Risk
Call report ไม่ครบ / ไม่ consistent — AI จะ generate ผิด
Mitigation: กำหนด mandatory fields ใน Phase 1 ก่อน build AI
RM Adoption
Medium Risk
RM มองว่า AI เป็นเครื่องมือ track performance ไม่ใช่ช่วยงาน
Mitigation: เน้น value ลด admin — demo ก่อน rollout
Data Security / Privacy
Medium Risk
ข้อมูลลูกค้า corporate ผ่าน 3rd-party API (Claude / Tavily)
Mitigation: ใช้ Azure OpenAI / Claude on Azure — data stays in region
Teams App Approval
Low Risk
IT security review ใช้เวลา 4–8 สัปดาห์ ทำให้ timeline ช้า
Mitigation: เริ่ม IT review ควบคู่กับ Phase 2 development
Recommended Starting Point: เริ่ม Phase 1 pilot กับ Corporate Banking RM 4 คน โดยไม่ต้อง build AI layer ก่อน — validate data quality และ adoption ก่อน ถ้า Phase 1 ผ่าน ROI จะชัดเจนพอสำหรับ board approval ใน Phase 2–3
